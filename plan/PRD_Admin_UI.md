# PRD: Nirvana DX — 관리자 페이지 UI

> **Product**: Myeongdong Kimbap 관리자 앱  
> **버전**: v1.0  
> **작성일**: 2026-05-24  
> **플랫폼**: 모바일 웹 (PWA, iOS/Android 브라우저 최적화)  
> **연관 문서**: [PRD_Frontend_UI.md](./PRD_Frontend_UI.md) — 고객용 메뉴 UI

## 1. 개요 (Overview)

### 1.1 배경
Nirvana DX 플랫폼의 음식점 관리자가 QR 접속 현황, 메뉴 구성, 주문 통계를 실시간으로 모니터링하고, 다국어 메뉴 정보를 직접 편집할 수 있는 모바일 관리자 전용 UI를 제공한다.

### 1.2 목표
- 메뉴 추가·수정·삭제를 관리자가 코드 없이 직접 수행
- QR 접속 통계 및 인기 메뉴 데이터를 한눈에 파악
- 다국어 메뉴 정보(한/영/일/중)를 단일 화면에서 일괄 관리
- 운영 현황을 실시간으로 파악하여 빠른 의사결정 지원

### 1.3 대상 사용자
| 사용자 유형 | 설명 |
|---|---|
| 매장 관리자 | 메뉴 편집, 통계 확인, 운영 현황 모니터링 |
| 매장 직원 | 주문 접수 현황 확인 (읽기 전용) |

### 1.4 범위 외 (Out of Scope)
- 결제 처리 및 POS 연동
- 재고 관리
- 다중 지점 관리 (v1.0)
- 데스크탑 전용 레이아웃 최적화

## 2. 전체 화면 구조 (Information Architecture)

```
관리자 앱
├── [Screen 1] 대시보드       — 실시간 KPI 요약 + 최근 활동
├── [Screen 2] 메뉴 관리      — 메뉴 목록 조회 + 카테고리 필터
│   └── [Screen 3] 메뉴 수정  — 다국어 정보 편집 + 이미지 업로드
├── [Screen 4] 주문           — 주문 접수 내역 (v1.0 기본 목록)
├── [Screen 5] 통계           — 접속·주문·필터 데이터 시각화
└── [Screen 6] 설정           — 매장 정보, 계정, 알림 (v1.0 기본)
```

### 하단 네비게이션 바 (공통)
| 탭 | 아이콘 | 화면 |
|---|---|---|
| 대시보드 | 격자 아이콘 | Screen 1 |
| 메뉴 | 포크+나이프 아이콘 | Screen 2 |
| 주문 | 쇼핑백 아이콘 | Screen 4 |
| 통계 | 막대 그래프 아이콘 | Screen 5 |
| 설정 | 톱니바퀴 아이콘 | Screen 6 |

- 활성 탭: 오렌지(`#E8622A`) 아이콘 + 텍스트
- 비활성 탭: 회색(`#888888`) 아이콘 + 텍스트

## 3. 화면별 요구사항 (Screen Requirements)

### 3.1 Screen 1 — 대시보드 (Dashboard)

#### 목적
오늘의 핵심 운영 지표와 최근 활동을 한눈에 파악한다.

#### UI 구성 요소

**헤더**
| 요소 | 설명 |
|---|---|
| 햄버거 메뉴 아이콘 | 사이드 메뉴 (v1.0 미구현 가능) |
| 매장 아이콘 + 매장명 | "명동김밥 관리자" |
| 알림 벨 아이콘 🔔 | 새 알림 시 빨간 뱃지 표시 |

**KPI 카드 영역 (2×2 그리드)**
| 카드 | 지표명 | 표시 형식 | 강조 색상 |
|---|---|---|---|
| 좌상단 | 오늘 QR 접속 수 | 숫자 + 전일 대비 퍼센트 | 오렌지 |
| 우상단 | 오늘 주문 수 | 숫자 + 전일 대비 퍼센트 | 오렌지 |
| 좌하단 | 주요 언어 | 언어명 + 접속 비율 % | 오렌지 |
| 우하단 | 인기 메뉴 | 메뉴명 + 주문 수 + 썸네일 | — |

- 전일 대비 증감: `+n%` 초록, `-n%` 빨강으로 방향 표시
- 카드 배경: 흰색(`#FFFFFF`), 둥근 모서리 12px, 그림자

**최근 활동 섹션**
- 섹션 제목 "최근 활동" + 우측 "전체 보기" 링크 (오렌지)
- 활동 목록: 최대 5건, 스크롤 없음
- 각 항목 구성:

| 요소 | 설명 |
|---|---|
| 활동 아이콘 | 활동 유형별 색상 아이콘 (아래 참조) |
| 활동 제목 | 굵은 폰트 (예: "새 주문 접수 #1024") |
| 활동 부제목 | 회색 서브텍스트 (예: "참치김밥 x1") |
| 경과 시간 | 우측 정렬, 회색 소형 텍스트 |

**활동 유형별 아이콘 색상**
| 활동 유형 | 아이콘 | 색상 |
|---|---|---|
| 새 주문 접수 | 쇼핑백 | 오렌지 |
| QR 스캔 | QR 코드 | 파란색 |
| 메뉴 업데이트 | 편집 | 초록색 |
| 정문 QR 스캔 | 위치 핀 | 보라색 |

#### 동작 요구사항
- KPI 데이터: 페이지 진입 시 자동 로드, 30초 간격 자동 새로고침
- 전일 대비 퍼센트: 소수점 첫째 자리 반올림 후 표시
- "전체 보기" 클릭 시 활동 전체 목록 화면으로 이동
- 인기 메뉴 카드의 썸네일 이미지: 원형 크롭(48×48px)

### 3.2 Screen 2 — 메뉴 관리 (Menu Management)

#### 목적
등록된 전체 메뉴를 조회하고 카테고리·검색으로 빠르게 탐색한다.

#### UI 구성 요소

**헤더**
| 요소 | 설명 |
|---|---|
| 햄버거 메뉴 아이콘 | 사이드 메뉴 |
| "메뉴 관리" 제목 | 가운데 정렬, 굵은 폰트 |
| 필터 아이콘 | 정렬 기준 선택 하단 시트 (인기순, 가격순, 이름순) |

**검색 바**
- 플레이스홀더: "메뉴 검색"
- 실시간 메뉴명 필터링
- 우측 X 버튼: 입력 초기화

**카테고리 필터 (수평 스크롤 필 형태)**
| 필 | 기본 상태 |
|---|---|
| 전체 | 선택(오렌지 배경 + 흰 텍스트) |
| 김밥 | 미선택(흰 배경 + 회색 테두리) |
| 라면 | 미선택 |
| 떡볶이 | 미선택 |
| 음료 | 미선택 |

- 선택된 카테고리 필 1개만 활성화 (단일 선택)
- 카테고리는 관리자 설정에서 추가·삭제 가능 (v1.1)

**메뉴 아이템 목록 (리스트 형태)**
| 요소 | 설명 |
|---|---|
| 썸네일 이미지 | 좌측, 정사각형 64×64px, 둥근 모서리 8px |
| 메뉴명 | 굵은 폰트, 한국어 기준 |
| 식단 태그 배지 | ⭐인기, ❌돼지고기 없음, 🌶매움, 🌿비건 등 (아래 규격 참조) |
| 가격 | ₩ 단위, 오렌지 색상 |
| 편집 아이콘 ✏️ | 우측, 터치 시 메뉴 수정 화면으로 이동 |

**식단 태그 배지 규격**
| 태그 | 아이콘 | 배경 | 텍스트 |
|---|---|---|---|
| 인기 | ⭐ | 노랑 `#FFF3CD` | `#856404` |
| 돼지고기 없음 | ❌ | 연빨강 `#FDECEA` | `#C62828` |
| 매움 | 🌶 | 연빨강 `#FDECEA` | `#C62828` |
| 비건 | 🌿 | 연초록 `#E8F5E9` | `#2E7D32` |
| 소고기 포함 | 🥩 | 연갈색 `#FBE9E7` | `#BF360C` |

**하단 CTA 버튼**
- 텍스트: "+ 메뉴 추가"
- 색상: 오렌지(`#E8622A`), 전체 너비, 높이 56px, 둥근 모서리 28px
- 터치 시 빈 폼의 메뉴 수정 화면으로 이동 (신규 등록 모드)

#### 동작 요구사항
- 검색 + 카테고리 필터 동시 적용 (AND 조건)
- 아이템 롱탭 시 삭제 컨텍스트 메뉴 표시
- 목록 당겨서 새로고침(Pull-to-Refresh) 지원
- 메뉴 없을 경우 빈 상태 일러스트 + "메뉴를 추가해 주세요" 안내 텍스트 표시

### 3.3 Screen 3 — 메뉴 수정 (Menu Edit)

#### 목적
선택한 메뉴의 다국어 이름, 가격, 이미지, 식단 정보를 편집하거나 신규 메뉴를 등록한다.

#### UI 구성 요소

**헤더**
| 요소 | 설명 |
|---|---|
| 뒤로가기 버튼 ← | 메뉴 관리 화면으로 복귀 (변경 미저장 시 확인 다이얼로그) |
| "메뉴 수정" / "메뉴 추가" 제목 | 모드에 따라 동적 표시 |
| 삭제 아이콘 🗑 | 해당 메뉴 삭제 (확인 다이얼로그 필수), 신규 등록 모드에서는 숨김 |

**이미지 업로드 영역**
| 요소 | 설명 |
|---|---|
| 이미지 미리보기 | 전체 너비, 세로 200px, object-fit: cover |
| 업로드 오버레이 | 이미지 없을 시 구름+화살표 아이콘 + "이미지 업로드" 텍스트 + "PNG, JPG (최대 5MB)" 안내 |
| 터치 시 동작 | 기기 갤러리 열기 (파일 선택) |

**다국어 이름 입력 필드**
| 언어 레이블 | 입력 필드 예시 |
|---|---|
| 한국어 | 참치김밥 |
| 영어 | Tuna Gimbap |
| 일본어 | ツナキンバ |
| 중국어(간체) | 金枪鱼紫菜包饭 |

- 각 필드: 라벨 + 1줄 텍스트 입력, 테두리 스타일 인풋
- 한국어 필드 필수 입력 (유효성 검사)

**가격 입력 필드**
- 레이블: "가격(원)"
- 좌측 ₩ 접두사 표시, 숫자 전용 키패드
- 값: 숫자만 허용, 최소 0원

**식단 정보 토글 (Toggle Switches)**
| 항목 | 아이콘 | 기본값 |
|---|---|---|
| 비건 | 🌿 | OFF |
| 돼지고기 없음 | ❌ | OFF |
| 소고기 포함 | 🥩 | OFF |
| 매움 | 🌶 | OFF |
| 인기 메뉴 | ⭐ | OFF |

- 토글 ON 상태: 오렌지 배경 (`#E8622A`)
- 토글 OFF 상태: 회색 배경 (`#CCCCCC`)
- 각 행: 좌측 아이콘 + 항목명, 우측 토글 스위치

**하단 CTA 버튼**
- 텍스트: "변경사항 저장" (수정 모드) / "메뉴 등록" (신규 모드)
- 색상: 오렌지(`#E8622A`), 전체 너비, 높이 56px, 둥근 모서리 28px
- 필수 필드 미입력 시 비활성화(`opacity: 0.4`), 클릭 불가

#### 동작 요구사항
- 이미지 파일 크기 초과(5MB) 시 토스트 오류 메시지 표시
- 지원 포맷 외 파일 선택 시 오류 메시지 표시 (PNG, JPG만 허용)
- 저장 성공 시 "저장되었습니다" 토스트 + 메뉴 관리 화면으로 복귀
- 저장 실패 시 "저장에 실패했습니다. 다시 시도해주세요" 토스트
- 뒤로가기 시 미저장 변경사항 있으면 "변경사항을 저장하지 않고 나가시겠습니까?" 확인 다이얼로그
- 삭제 버튼 클릭 시 "이 메뉴를 삭제하시겠습니까?" 확인 다이얼로그 (취소 / 삭제 버튼)

### 3.4 Screen 4 — 통계 (Statistics)

#### 목적
QR 접속 패턴, 주문 데이터, 필터 사용 현황을 시각화하여 운영 인사이트를 제공한다.

#### UI 구성 요소

**헤더**
| 요소 | 설명 |
|---|---|
| 햄버거 메뉴 아이콘 | 사이드 메뉴 |
| "통계" 제목 | 가운데 정렬 |
| 날짜 선택기 📅 | "오늘 ▾" 드롭다운 (오늘 / 이번 주 / 이번 달 / 직접 선택) |

**섹션 1: 언어별 접속 현황**
- 섹션 제목: "언어별 접속 현황"
- 도넛 차트 (좌측) + 범례 목록 (우측)
- 범례 항목: 색상 점 + 언어명 + 비율 %

| 언어 | 색상 |
|---|---|
| 영어 | 오렌지 `#E8622A` |
| 일본어 | 빨강 `#F44336` |
| 중국어 | 파란-청록 `#26C6DA` |
| 한국어 | 초록 `#66BB6A` |

**섹션 2: 인기 메뉴 TOP**
- 섹션 제목: "인기 메뉴 TOP"
- 순위 목록 (최대 3건)
- 각 항목 구성:

| 요소 | 설명 |
|---|---|
| 순위 번호 | 굵은 폰트 (1, 2, 3) |
| 썸네일 이미지 | 원형, 40×40px |
| 메뉴명 | 굵은 폰트 |
| 수평 바 차트 | 1위 기준 상대적 너비, 오렌지 색상 |
| 주문 수 | 우측 정렬 숫자 |

**섹션 3: QR 위치별 접속**
- 섹션 제목: "QR 위치별 접속"
- 도넛 차트 (좌측) + 범례 목록 (우측)

| QR 위치 | 색상 |
|---|---|
| 정문 QR | 오렌지 `#E8622A` |
| 테이블 QR | 파랑 `#42A5F5` |
| 기타 | 초록 `#66BB6A` |

**섹션 4: 필터 사용 비율**
- 섹션 제목: "필터 사용 비율"
- 가로 배열 배지 형태: 아이콘 + 태그명 + 비율 %

| 필터 항목 | 아이콘 | 예시 비율 |
|---|---|---|
| 돼지고기 없음 | ❌ | 38% |
| 매움 | 🌶 | 32% |
| 비건 | 🌿 | 18% |
| 소고기 포함 | 🥩 | 12% |

#### 동작 요구사항
- 날짜 선택기 변경 시 모든 섹션 데이터 일괄 갱신
- 차트 터치 시 해당 항목 수치 툴팁 표시
- 데이터 로딩 중 스켈레톤 UI 표시
- 데이터 없음 상태: 섹션별 "데이터가 없습니다" 안내 텍스트

## 4. 공통 컴포넌트 (Shared Components)

### 4.1 토스트 메시지
- 위치: 화면 하단, 하단 네비게이션 바 위 16px
- 성공: 초록 배경 `#4CAF50` + 흰 텍스트 + ✓ 아이콘
- 오류: 빨강 배경 `#F44336` + 흰 텍스트 + ✕ 아이콘
- 지속 시간: 2.5초 자동 소멸
- 진입/퇴장 애니메이션: slide-up / fade-out

### 4.2 확인 다이얼로그 (Confirm Dialog)
- 오버레이: 반투명 검정 배경 (`rgba(0,0,0,0.5)`)
- 카드: 흰 배경, 둥근 모서리 16px, 가운데 정렬
- 구성: 제목 + 내용 텍스트 + [취소] [확인] 버튼
- 취소 버튼: 회색 텍스트
- 확인/삭제 버튼: 오렌지 또는 빨강 강조

### 4.3 스켈레톤 UI
- 데이터 로딩 시 실제 레이아웃과 동일한 회색 블록 표시
- 애니메이션: shimmer (좌→우 반짝임)

### 4.4 빈 상태 (Empty State)
- 일러스트 아이콘 + 안내 텍스트 + (필요 시) 액션 버튼
- 예: 메뉴 없음 → "+ 메뉴 추가하기" 버튼 표시

## 6. 인증 및 보안 (Authentication & Security)

### 6.1 접근 제어
- 관리자 앱 진입 시 로그인 필수 (JWT 기반)
- 세션 만료: 24시간 비활동 시 자동 로그아웃
- 로컬 스토리지에 민감 정보 저장 금지 (토큰은 httpOnly 쿠키 또는 메모리 관리)

### 6.2 권한 구분
| 권한 | 가능한 작업 |
|---|---|
| 관리자 (admin) | 메뉴 CRUD, 통계 조회, 설정 변경 |
| 직원 (staff) | 주문 현황 조회 (읽기 전용) |

### 6.3 입력 유효성 검사
- 이미지 업로드: 파일 타입(MIME type) 서버 측 검증, 크기 제한 5MB 강제
- 가격 필드: 음수 및 비숫자 입력 차단
- 메뉴명: XSS 방지 위한 HTML 특수문자 이스케이프 처리

## 7. 반응형 및 접근성

### 7.1 반응형
- 타깃: 360px ~ 430px (모바일 우선)
- 태블릿(768px+): 중앙 정렬 + 최대 너비 430px 제한
- 데스크탑 레이아웃 최적화는 v1.1에서 검토

### 7.2 접근성 (WCAG 2.1 AA)
- 모든 이미지 `alt` 속성 필수
- 색상 대비비 4.5:1 이상 (텍스트 기준)
- 터치 영역 최소 44×44px
- 포커스 인디케이터 명확히 표시
- 토글 스위치: `aria-checked`, `role="switch"` 속성 적용

## 8. API 연동 요구사항 (Frontend Perspective)

### 8.1 대시보드
| 데이터 | 엔드포인트 (예시) | 갱신 주기 |
|---|---|---|
| 오늘 KPI 요약 | `GET /api/admin/dashboard/summary` | 30초 폴링 또는 WebSocket |
| 최근 활동 목록 | `GET /api/admin/activities?limit=5` | 동일 |

### 8.2 메뉴 관리
| 작업 | 엔드포인트 (예시) | 비고 |
|---|---|---|
| 메뉴 목록 조회 | `GET /api/admin/menus?category=&keyword=` | 쿼리 파라미터 필터 |
| 메뉴 단건 조회 | `GET /api/admin/menus/:id` | 수정 화면 진입 시 |
| 메뉴 등록 | `POST /api/admin/menus` | multipart/form-data (이미지 포함) |
| 메뉴 수정 | `PATCH /api/admin/menus/:id` | multipart/form-data |
| 메뉴 삭제 | `DELETE /api/admin/menus/:id` | 소프트 삭제 권장 |

### 8.3 통계
| 데이터 | 엔드포인트 (예시) |
|---|---|
| 언어별 접속 통계 | `GET /api/admin/stats/language?from=&to=` |
| 인기 메뉴 TOP | `GET /api/admin/stats/popular-menus?from=&to=&limit=3` |
| QR 위치별 접속 | `GET /api/admin/stats/qr-location?from=&to=` |
| 필터 사용 비율 | `GET /api/admin/stats/filter-usage?from=&to=` |

## 9. 에러 처리 및 엣지 케이스

| 서버 오류 (5xx) | "서버 오류가 발생했습니다 (에러 코드)" 토스트 |
