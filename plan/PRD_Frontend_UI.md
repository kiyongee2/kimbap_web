# PRD: Nirvana DX — Multilingual QR Menu Frontend UI

> **Product**: Myeongdong Kimbap QR 메뉴 주문 앱  
> **버전**: v1.0  
> **작성일**: 2026-05-24  
> **플랫폼**: 모바일 웹 (PWA, iOS/Android 브라우저 최적화)

---

## 1. 개요 (Overview)

### 1.1 배경
외국인 방문객이 많은 명동 지역 김밥 음식점에서 QR 코드 스캔 한 번으로 다국어 메뉴 탐색 및 주문이 가능한 모바일 웹 UI를 제공한다. 직원 호출 없이 고객이 직접 메뉴를 확인하고 주문 요약 화면을 직원에게 보여주는 방식으로 운영한다.

### 1.2 목표
- 언어 장벽 없이 누구나 3분 이내에 주문 완료
- 식단 정보(알레르기, 채식 등) 명확한 시각적 표시
- 직원-고객 간 주문 오류 최소화

### 1.3 대상 사용자
| 사용자 유형 | 설명 |
|---|---|
| 외국인 관광객 | 영어, 일본어, 중국어 사용자 |
| 내국인 고객 | 한국어 사용자 |
| 음식점 직원 | 주문 확인 수신자 (별도 UI 없음) |

---

## 2. 사용자 플로우 (User Flow)

```
QR 코드 스캔
    ↓
[Screen 1] 언어 선택
    ↓
[Screen 2] 메뉴 목록 (검색 / 필터)
    ↓
[Screen 3] 메뉴 상세
    ↓ (수량 선택 → 장바구니 추가)
[Screen 4] 주문 요약 (직원에게 화면 제시)
    ↓
주문 완료
```

---

## 3. 화면별 요구사항 (Screen Requirements)

### 3.1 Screen 1 — 언어 선택 (Language Selection)

#### 목적
진입 시 사용자가 원하는 언어를 선택하면 이후 모든 UI 텍스트가 해당 언어로 표시된다.

#### UI 구성 요소
| 요소 | 설명 |
|---|---|
| 브랜드 로고 | 김밥 일러스트 + 매장명 "Myeongdong Kimbap" |
| 환영 메시지 | "Welcome to Myeongdong Kimbap" + 서브 문구 |
| 언어 선택 버튼 | English(기본 강조), 日本語, 中文, 한국어 |
| 안내 문구 | "Please select your language" |

#### 동작 요구사항
- 각 언어 버튼에 국기/언어 아이콘 표시
- 선택된 언어 버튼은 오렌지(`#E8622A`) 배경 + 흰색 텍스트로 강조
- 미선택 버튼은 흰색 배경 + 테두리 스타일
- 선택 즉시 메뉴 목록 화면으로 전환 (애니메이션: slide-up)
- 선택한 언어는 세션 동안 유지 (localStorage)

#### 지원 언어
`en` | `ja` | `zh` | `ko`

---

### 3.2 Screen 2 — 메뉴 목록 (Menu List)

#### 목적
전체 메뉴를 탐색하고 필터/검색으로 원하는 메뉴를 빠르게 찾는다.

#### UI 구성 요소

**헤더**
| 요소 | 설명 |
|---|---|
| 햄버거 메뉴 아이콘 | 사이드 메뉴 또는 카테고리 네비게이션 |
| 매장명 | "Myeongdong Kimbap" |
| 언어 변경 버튼 | 현재 언어 표시 + 드롭다운 (예: 🌐 English ▾) |

**검색 바**
- 플레이스홀더: "Search menu"
- 실시간 텍스트 필터링 (메뉴명, 재료명 기준)

**필터 태그 (복수 선택 가능)**
| 태그 | 아이콘 색상 | 의미 |
|---|---|---|
| Vegan | 초록 🌿 | 채식 메뉴 |
| No Pork | 빨강 🚫 | 돼지고기 없음 |
| Beef | 갈색 🥩 | 소고기 포함 |
| Spicy | 빨강 🌶 | 매운 음식 |
| Popular | 노랑 ⭐ | 인기 메뉴 |

**메뉴 카드 (리스트 형태)**
| 요소 | 설명 |
|---|---|
| 썸네일 이미지 | 정사각형, 좌측 배치 (80×80px) |
| 메뉴명 | 굵은 폰트, 선택 언어 기준 |
| 재료 설명 | 서브 텍스트, 회색 |
| 식단 태그 | No Pork / Spicy / Popular 등 배지 |
| 가격 | 우측 정렬, 오렌지 색상 (₩ 단위) |

**하단 네비게이션 바**
| 탭 | 아이콘 |
|---|---|
| Menu | 포크+나이프 아이콘 |
| Order | 쇼핑백 아이콘 |
| Info | ⓘ 아이콘 |
| Language | 🌐 아이콘 |

#### 동작 요구사항
- 필터 태그 선택 시 해당 조건에 맞는 메뉴만 표시 (AND 조건)
- 검색 + 필터 동시 적용 가능
- 메뉴 카드 터치 시 메뉴 상세 화면으로 전환
- Order 탭 배지: 장바구니 담긴 수량 표시 (예: 빨간 숫자 뱃지)

---

### 3.3 Screen 3 — 메뉴 상세 (Menu Detail)

#### 목적
선택한 메뉴의 상세 정보 확인 및 장바구니 추가.

#### UI 구성 요소

**헤더**
| 요소 | 설명 |
|---|---|
| 뒤로가기 버튼 | ← 아이콘, 메뉴 목록으로 복귀 |
| 즐겨찾기 버튼 | ♡ 아이콘 (우측 상단), 토글 시 채워진 하트로 변경 |

**이미지 영역**
- 전체 너비 이미지 슬라이더 (여러 장 지원)
- 하단 페이지네이션 인디케이터 (점 형태)

**메뉴 정보**
| 요소 | 설명 |
|---|---|
| 메뉴명 | 큰 제목 폰트 |
| 가격 | 메뉴명 우측, 오렌지 색상 |
| 재료 설명 | 메뉴명 하단 소형 텍스트 |

**식단 정보 (Dietary Info)**
배지 형태로 아이콘 + 텍스트 표시:
| 배지 | 색상 |
|---|---|
| Fish | 파란 계열 |
| Egg | 노란 계열 |
| No Pork | 빨간 계열 |
| Not Spicy | 초록 계열 |

**수량 선택 (Quantity)**
- `−` 버튼 | 수량 숫자 | `+` 버튼
- 최소 수량: 1 (0 이하 불가)
- 최대 수량: 99

**하단 CTA 버튼**
- 텍스트: "Add to Order" + 장바구니 아이콘
- 색상: 오렌지(`#E8622A`), 전체 너비, 높이 56px, 둥근 모서리(28px)

#### 동작 요구사항
- 이미지 슬라이더: 좌우 스와이프 지원
- "Add to Order" 클릭 시 장바구니에 항목 추가 후 성공 토스트 표시
- 즐겨찾기 상태는 localStorage에 저장

---

### 3.4 Screen 4 — 주문 요약 (Order Summary)

#### 목적
담긴 메뉴 최종 확인 후 직원에게 화면을 제시해 주문을 완료한다.

#### UI 구성 요소

**헤더**
| 요소 | 설명 |
|---|---|
| "Your Order" 제목 | 굵은 폰트 |
| 삭제 아이콘 🗑 | 전체 주문 초기화 |

**주문 항목 목록**
| 요소 | 설명 |
|---|---|
| 썸네일 이미지 | 좌측 배치 |
| 메뉴명 + 수량 | 예: "Tuna Gimbap x1" |
| 금액 | 우측 정렬 |

**합계**
- "Total" + 총 금액 (굵은 폰트)

**직원 제시 영역 (Staff Display Panel)**
- 오렌지 점선 테두리 박스
- 상단: 직원 아이콘 + "Show this screen to staff" (선택 언어)
- 하단: 주문 내용을 **한국어로** 크게 표시 (예: 참치김밥 1개 / 라면 1개)
- 목적: 언어와 무관하게 직원이 주문 내용을 이해할 수 있도록

**확인 버튼**
- 텍스트: "Confirm" + ✓ 아이콘
- 색상: 오렌지(`#E8622A`), 전체 너비

#### 동작 요구사항
- 항목별 수량 수정 가능 (롱탭 또는 스와이프 삭제)
- 전체 삭제 시 확인 다이얼로그 표시
- "Confirm" 클릭 시 주문 완료 화면으로 전환 (장바구니 초기화)
- 직원 제시 영역의 한국어 텍스트는 폰트 크기 최소 28px 이상

## 4. 디자인 시스템 (Design System)

### 4.1 컬러 팔레트
| 토큰 | 색상 코드 | 용도 |
|---|---|---|
| `--color-primary` | `#E8622A` | CTA 버튼, 강조 텍스트, 활성 탭 |
| `--color-background` | `#FDF6EE` | 전체 배경 (크림색) |
| `--color-surface` | `#FFFFFF` | 카드, 모달 배경 |
| `--color-text-primary` | `#1A1A1A` | 본문 텍스트 |
| `--color-text-secondary` | `#888888` | 서브 텍스트, 설명 |
| `--color-border` | `#E0E0E0` | 테두리, 구분선 |
| `--color-tag-vegan` | `#4CAF50` | Vegan 태그 |
| `--color-tag-spicy` | `#F44336` | Spicy 태그 |
| `--color-tag-popular` | `#FFC107` | Popular 태그 |

### 4.2 타이포그래피
| 스타일 | 크기 | 굵기 | 용도 |
|---|---|---|---|
| Display | 28px | 700 | 브랜드명, 메뉴 제목 |
| Heading | 20px | 600 | 섹션 헤더 |
| Body | 14px | 400 | 재료 설명, 본문 |
| Caption | 12px | 400 | 부가 정보 |
| Price | 16px | 700 | 가격 표시 |
| Staff Korean | 28px+ | 700 | 직원 제시 한국어 텍스트 |

### 4.3 간격 및 레이아웃
- 기본 패딩: 16px (좌우)
- 카드 간격: 12px
- 버튼 높이: 48px (일반), 56px (CTA)
- 모서리 반경: 12px (카드), 28px (전체 너비 버튼), 20px (태그)
- 최대 너비: 430px (모바일 중심)

### 4.4 아이콘
- 스타일: Outlined (비활성) / Filled (활성)
- 크기: 24px (네비게이션), 20px (인라인)
- 라이브러리: Material Symbols 또는 커스텀 SVG

## 5. 다국어 처리 (i18n)

### 5.1 지원 언어 및 우선순위
1. 영어 (en) — 기본값
2. 일본어 (ja)
3. 중국어 간체 (zh)
4. 한국어 (ko)

### 5.2 번역 구조
```json
{
  "en": {
    "welcome": "Welcome to",
    "selectLanguage": "Please select your language",
    "searchPlaceholder": "Search menu",
    "addToOrder": "Add to Order",
    "yourOrder": "Your Order",
    "showToStaff": "Show this screen to staff",
    "confirm": "Confirm",
    "total": "Total",
    "quantity": "Quantity",
    "dietaryInfo": "Dietary Info"
  }
}
```

### 5.3 번역 규칙
- UI 텍스트 전체는 선택 언어로 표시
- 주문 요약의 "직원 제시 영역"은 **항상 한국어** 고정
- 메뉴명과 재료 설명은 DB에서 언어별 필드로 관리
- RTL(아랍어 등) 미지원 (v1.0 범위 외)

## 6. 반응형 및 접근성

### 6.1 반응형
- 타깃: 360px ~ 430px 너비 (모바일 우선)
- 태블릿(768px+): 중앙 정렬 + 최대 너비 제한
- 데스크탑 지원 불필요 (QR 스캔 기반 모바일 전용)

### 6.2 접근성 (WCAG 2.1 AA 준수)
- 모든 이미지에 `alt` 속성 필수
- 색상 대비비 4.5:1 이상 (텍스트 기준)
- 터치 영역 최소 44×44px
- 포커스 인디케이터 명확히 표시
- 스크린 리더 대응 (`aria-label` 적용)
