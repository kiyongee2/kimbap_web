## 1. 개요 (Overview)

### 1.1 배경
관리자 페이지의 주문 내역 화면에서 주문이 누적될수록 최신 주문을 빠르게 파악하기 어렵고, 서버의 UTC 타임스탬프가 그대로 노출되어 매장 운영자에게 혼란을 주었다.  
또한 주문 수가 많을 경우 한 화면에 모든 주문이 나열되어 가독성이 저하되었다.

### 1.2 목표
- 최신 주문이 항상 목록 상단에 표시되도록 내림차순 정렬 적용
- 모든 시간 표시를 대한민국 표준시(KST, UTC+9)로 통일
- 주문 목록을 5개 단위로 페이지 분할하여 가독성 및 스크롤 피로도 개선

### 1.3 대상 사용자
| 사용자 유형 | 설명 |
|---|---|
| 매장 관리자 | 주문 접수 현황 실시간 모니터링 |
| 매장 직원 | 주문 내역 조회 (읽기 전용) |

### 1.4 범위 외 (Out of Scope)
- 백엔드 정렬 API 변경 (프론트엔드에서 처리)
- 타임존 설정 변경 기능 (KST 고정)
- 페이지 크기 사용자 설정

## 2. 기능 요구사항

### 2.1 내림차순 정렬 (최신 주문 우선)

| 항목 | 내용 |
|---|---|
| **기능 ID** | ORD-SORT-01 |
| **우선순위** | High |
| **설명** | 주문 목록을 `createdAt` 필드 기준 내림차순으로 정렬하여 가장 최근 주문이 첫 번째에 표시된다 |
| **트리거** | 목록 최초 로드, 5초 자동 갱신, 새로고침(🔄) 버튼 클릭 |
| **정렬 기준** | `createdAt` (ISO 8601 타임스탬프) 내림차순 |
| **처리 위치** | 프론트엔드 (API 응답 수신 후 클라이언트 정렬) |

**수용 기준 (Acceptance Criteria)**
- [ ] 페이지 진입 시 가장 최근 주문이 목록 1번째 카드에 표시된다
- [ ] 5초 자동 갱신 후에도 내림차순 정렬 순서가 유지된다
- [ ] 새로고침 버튼 클릭 후에도 내림차순 정렬 순서가 유지된다
- [ ] 동일 `createdAt`인 경우 순서는 API 응답 순서를 따른다

---

### 2.2 대한민국 표준시(KST) 시간 표시

| 항목 | 내용 |
|---|---|
| **기능 ID** | ORD-TIME-01 |
| **우선순위** | High |
| **설명** | 주문 카드의 시간 표시를 KST(UTC+9) 기준으로 변환하여 `M월 D일 HH:MM` 형식으로 출력한다 |
| **표시 형식** | `ko-KR` 로케일, `timeZone: 'Asia/Seoul'`, 월·일·시·분 표시 |
| **처리 위치** | 프론트엔드 (`formatDateTime` 함수) |

**표시 형식 예시**
```
서버 저장값 (UTC): 2026-06-04T01:30:00Z
화면 표시 (KST):  6월 4일 오전 10:30
```

**수용 기준 (Acceptance Criteria)**
- [ ] UTC 타임스탬프가 KST(+9시간)로 변환되어 표시된다
- [ ] 클라이언트의 시스템 로케일에 관계없이 항상 `ko-KR` 형식으로 표시된다
- [ ] 자정 전후(예: UTC 15:00 → KST 다음날 00:00)도 정확히 변환된다

---

### 2.3 페이지네이션 (5개 단위)

| 항목 | 내용 |
|---|---|
| **기능 ID** | ORD-PAGE-01 |
| **우선순위** | Medium |
| **설명** | 주문 목록을 5개씩 나누어 페이지 단위로 표시하고, 이전/다음 버튼으로 탐색한다 |
| **페이지 크기** | 5개 (고정) |
| **UI 위치** | 주문 카드 목록 하단 |

**UI 구성**
```
[ ◀ ]   2 / 5   [ ▶ ]
```

| 컴포넌트 | 설명 |
|---|---|
| ◀ 버튼 | 이전 페이지 이동. 1페이지에서 비활성화(disabled) |
| 페이지 표시 | `현재 페이지 / 전체 페이지` 형식 |
| ▶ 버튼 | 다음 페이지 이동. 마지막 페이지에서 비활성화(disabled) |

**수용 기준 (Acceptance Criteria)**
- [ ] 주문이 5개 이하인 경우 페이지네이션 UI가 표시되지 않는다 (단일 페이지)
- [ ] 주문이 6개 이상인 경우 페이지네이션 UI가 하단에 표시된다
- [ ] 1페이지에서 ◀ 버튼이 비활성화된다
- [ ] 마지막 페이지에서 ▶ 버튼이 비활성화된다
- [ ] 새로고침(🔄) 버튼 클릭 시 1페이지로 초기화된다
- [ ] 5초 자동 갱신 시 현재 페이지가 유지된다
- [ ] 페이지 이동 후에도 내림차순 정렬 순서가 유지된다

## 3. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| **성능** | 정렬 및 페이지 슬라이싱은 클라이언트에서 처리하여 추가 API 호출 없음 |
| **반응성** | 페이지 전환 즉시 반응 (네트워크 요청 없음) |
| **접근성** | ◀/▶ 버튼에 `disabled` 속성 적용하여 스크린리더 대응 |
| **일관성** | 자동 갱신(5초) 중 페이지 전환이 발생해도 현재 페이지 유지 |
