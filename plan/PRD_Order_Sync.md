## 1. 개요 (Overview)

### 1.1 배경
고객 앱(`/`)과 관리자 앱(`/admin`)은 동일한 단일 페이지 애플리케이션(SPA) 번들에서 라우팅으로 분리된 두 개의 React 루트이다.  
두 루트는 서로 다른 탭에서 실행되므로 React 상태를 직접 공유할 수 없다.  
고객이 주문을 확정하면 관리자 앱에 즉시 반영되어 매장이 주문을 인식할 수 있어야 한다.

### 1.2 목표
- 고객이 주문 확정 시 관리자 대시보드의 "오늘 주문" 카운트가 실시간으로 증가
- 관리자 "주문" 탭에서 각 주문의 상세 내역(메뉴명, 수량, 금액, 언어, 시각)을 조회
- 별도 서버 또는 WebSocket 없이 브라우저 내 localStorage만으로 동기화

### 1.3 대상 사용자
| 사용자 유형 | 역할 |
|---|---|
| 고객 | 메뉴 선택 후 주문 확정 → 데이터 생성자 |
| 매장 관리자/직원 | 관리자 앱에서 주문 현황 조회 → 데이터 소비자 |

### 1.4 범위 외 (Out of Scope)
- 서버 기반 실시간 동기화 (WebSocket, SSE, Firebase 등)
- 다중 기기 간 실시간 공유 (서버 없이 불가)
- 주문 상태 변경 (접수 → 조리 중 → 완료) 워크플로우
- 결제 처리

## 2. 아키텍처 (Architecture)

### 2.1 전체 흐름

```
[ 고객 탭 ]                            [ 관리자 탭 ]
  고객 앱 (/)                            관리자 앱 (/admin)
     │                                        │
     │  주문 확정                              │
     ▼                                        │
 saveOrder(cart, lang)                        │
     │                                        │
     ▼                                        │
 localStorage                   ◄─────────── getOrders()
  kimbap_orders (JSON)           storage 이벤트 또는 5초 폴링
  kimbap_order_counter
```

### 2.2 저장소 설계

**브라우저 localStorage 키**

| 키 | 타입 | 설명 |
|---|---|---|
| `kimbap_orders` | JSON 배열 | 주문 목록 (최신순, 최대 100건) |
| `kimbap_order_counter` | 숫자 문자열 | 주문 일련번호 자동 증가 카운터 (시작값 1001) |

**주문 오브젝트 스키마**

```jsonc
{
  "id": 1001,                        // 자동 증가 주문 번호 (정수)
  "timestamp": "2026-05-25T10:30:00.000Z",  // ISO 8601 UTC
  "lang": "ko",                      // 고객 선택 언어: "ko" | "en" | "ja" | "zh"
  "items": [
    {
      "id": "menu-1",                // 메뉴 ID
      "name": { "ko": "참치김밥", "en": "Tuna Kimbap", "ja": "ツナキンパ", "zh": "金枪鱼紫菜卷" },
      "price": 4500,                 // 단가 (원)
      "quantity": 2,                 // 수량
      "emoji": "🍱"                  // 메뉴 이모지
    }
  ],
  "total": 9000                      // 합계 금액 = Σ(price × quantity)
}
```

### 2.3 공유 유틸리티 모듈

파일 경로: `ui/src/store/orderStore.js`

| 함수 | 서명 | 설명 |
|---|---|---|
| `saveOrder` | `(cart, lang) → Order` | 주문을 localStorage에 저장하고 저장된 객체 반환 |
| `getOrders` | `() → Order[]` | 전체 주문 목록 반환 (오류 시 빈 배열) |
| `getTodayOrders` | `() → Order[]` | 오늘 날짜의 주문만 필터링하여 반환 |

#### 동작 흐름

| ID | 요구사항 |

## 4. 동기화 메커니즘 상세

### 4.1 이벤트 기반 동기화 (다른 탭)

브라우저의 `window.storage` 이벤트는 **자신이 아닌 다른 탭/창**에서 같은 도메인의 localStorage가 변경될 때 발생한다.

```
고객 탭 (Tab A)          관리자 탭 (Tab B)
    │                          │
    │  localStorage.setItem()  │
    │ ─────────────────────►  storage 이벤트 발생
    │                          │
    │                     setOrders(getOrders())
    │                     → React 리렌더링
```

### 4.2 폴링 기반 동기화 (같은 탭)

같은 탭에서 경로 이동(`/` → `/admin`) 시에는 `storage` 이벤트가 발생하지 않는다.  
`setInterval(refresh, 5000)`으로 5초마다 localStorage를 재조회하여 보완한다.

### 4.3 초기 데이터 로딩

`useState(() => getOrders())` 형태의 지연 초기화(lazy initializer)로 컴포넌트 마운트 시 즉시 저장된 주문을 불러온다.

### 4.4 데이터 보존 정책

| 항목 | 정책 |
|---|---|
| 최대 보관 건수 | 100건 (초과 시 오래된 주문 자동 삭제) |
| 저장 단위 | 전체 배열을 단일 JSON 문자열로 직렬화 |
| 만료 정책 | 없음 (브라우저 localStorage는 명시적 삭제 전까지 유지) |

## 5. 비기능 요구사항

| ID | 항목 | 요구사항 |
|---|---|---|
| NFR-01 | 성능 | `getOrders()` 호출은 JSON.parse 포함 5ms 이내 완료 |
| NFR-02 | 안정성 | JSON 파싱 오류 발생 시 빈 배열 반환으로 앱 크래시 방지 |
| NFR-03 | 용량 | 주문 100건 기준 localStorage 사용량 약 50KB 이하 |
| NFR-04 | 메모리 | 컴포넌트 언마운트 시 이벤트 리스너 및 타이머 반드시 해제 |
| NFR-05 | 보안 | localStorage는 동일 출처(Same-Origin) 정책에 의해 도메인 외부 접근 차단 |

## 6. 제약사항 및 한계

| 제약 | 설명 |
|---|---|
| 다중 기기 공유 불가 | localStorage는 같은 브라우저 내에서만 공유. 관리자가 다른 기기에서 접속 시 동기화 안 됨 |
| 프라이빗 브라우징 | 시크릿/인코그니토 모드에서는 탭 간 localStorage 공유가 제한될 수 있음 |
| 브라우저 종료 시 | localStorage는 유지되지만, 진행 중 폴링/이벤트 리스너는 초기화됨 |
| 최대 저장 용량 | 브라우저 도메인당 localStorage는 통상 5~10MB. 장기 운영 시 정기 정리 필요 |

## 7. 향후 개선 방향 (Future Scope)

| 항목 | 설명 |
|---|---|
| 서버 연동 | REST API 또는 WebSocket으로 교체하여 다중 기기 실시간 동기화 지원 |
| 주문 상태 관리 | 접수 → 조리 중 → 완료 → 픽업 단계별 상태 추적 |
| 주문 알림 | 새 주문 접수 시 브라우저 Push Notification 또는 진동 알림 |
| 통계 연동 | 실제 주문 데이터를 통계 화면(Screen 5)에 반영 |
| 데이터 내보내기 | 일별/주별 주문 내역 CSV 다운로드 |
| 주문 취소 | 관리자가 주문 건을 취소 처리하고 고객에게 상태 표시 |
